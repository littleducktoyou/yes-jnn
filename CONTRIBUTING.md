# Contributing to yes-jnn

yes-jnn is licensed under AGPL-3.0 & developed primarily on [Lovable](https://lovable.dev). The `main` branch is public and synced bidirectionally with the Lovable project: commits pushed here appear in Lovable's editor, and changes made in Lovable get pushed here.

## Git history rules

> [!IMPORTANT]
> Don't force-push, rebase, or amend commits already on `main`. Rewriting published history corrupts Lovable's sync and permanently loses the project timeline on their side.

Use `git revert` if you need to undo something already pushed.

## Getting started

Clone the repo & run it locally by following [`BUILD.md`](./BUILD.md). It covers prerequisites, environment variables, & database setup.

## Pull request guidelines

- One feature or fix per PR; don't bundle unrelated changes
- Run `bun run build` before opening a PR; the CI gate won't merge a PR that fails to build
- Keep `main` in a working state at all times — Lovable pulls from it on every sync

## Project layout

```
src/
  components/       # UI components: NotesApp, LoginScreen, MarkdownToolbar
  components/ui/    # shadcn/ui primitives — don't edit; managed by Lovable
  hooks/            # React hooks (useAuth)
  integrations/     # Supabase client files — auto-generated; don't edit
  lib/              # Utility functions & error reporting
  routes/           # TanStack Router file-based routes (see src/routes/README.md)
supabase/
  migrations/       # SQL migration files — run these to set up a fresh database
```

## Auto-generated files

These three paths are regenerated automatically and will be overwritten on the next build or schema change. Don't edit them by hand:

- `src/integrations/supabase/` — regenerated when the Supabase schema changes in Lovable
- `src/routeTree.gen.ts` — regenerated by TanStack Router on every `bun run dev` or `bun run build`
- `src/components/ui/` — shadcn/ui components managed by Lovable

## Setting up your own Supabase project

1. Create a project at [supabase.com](https://supabase.com)
2. Run the migration in `supabase/migrations/` via the SQL Editor or `supabase db push`
3. Copy your Project URL & anon key into `.env` (see `.env.example` for variable names)
